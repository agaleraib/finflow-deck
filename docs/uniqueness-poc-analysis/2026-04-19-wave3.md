## Anomalies and caveats

### 1. Sample size: 1 same-variant pair per event

The persona-fixture distribution (a=v1, b=v2, c=v3, d=v1) gives only ONE same-variant pair per event (premium ↔ northbridge, both v1). With n=1, the same-variant baseline is essentially a single noisy data point. The cosine deltas above should be read as directional evidence, not confidence-intervalled measurements. To strengthen: either add more brokers (e.g. e=v2, f=v3) or add events that include cross-variant baseline pairs.

### 2. Fed-rate fidelity outlier (premium↔fasttrade, fid=0.75)

The judge flagged this pair `fabrication_risk` and gave it 0.75 fidelity. Inspecting `judgeFactualDivergences`:

- Document A (premium): "1.0650–1.0750 over the next two to four weeks" (range, no probability)
- Document B (fasttrade): "60% probability to a grind toward 1.0700" (point estimate + probability)
- Plus framing differences: A warns of crowded-trade short-squeeze risk; B presents the trade as accelerating with no countervailing caveat

These are forward-guidance / probability-assignment differences — not invented levels or fabricated data points (the underlying Fed numbers, rate spreads, and dot-plot facts are shared between both). The risk is **persona-driven** (fasttrade-pro's "salesman register" tone with confident probability assertions vs premium-capital-markets' measured advisory voice), not variant-driven (variant 2's structural template doesn't ask for probability assignments). Mitigation should target the persona prompt (constraining how fasttrade-pro frames forward views) rather than the variant.

### 3. broker-c v3 clamps to v1 on 2-variant identities

Spec §2.3 documented clamping: helix-markets (v3) on newsletter-editor or beginner-blogger would resolve to v1. Stage 6 only ran in-house-journalist (3-variant identity), so no clamping affected this wave. If future runs widen Stage 6 to multiple identities, expect the v3-clamp pairs to look identical to v1-baseline pairs and inflate the same-variant bucket artificially.

### 4. Editorial memory injection

Both runs logged `✓ injected editorial memory context for 4/4 personas` and `✓ recording 4 Stage 6 output(s) to editorial memory`. Backend confirmed Postgres on both events (`PostgresEditorialMemoryStore (postgresql://***@10.1.10.230:5432/finflow_dev)`). Editorial memory is exercising the production code path, not the in-memory fallback.

### 5. Stage 6 covers ONE identity, not all 6

The `crossTenantMatrix` in `raw-data.json` only contains in-house-journalist outputs. Per the runner, Stage 6 runs the load-bearing test on a single identity. If we want variant deltas across all 6 identities, that's a runner expansion (out of scope for this wave per spec §9 constraint on PoC scope, but worth flagging — the structural-variants test surface is currently 6× thinner than the persona-fixture distribution suggests).

### 6. fed-rate cross-tenant cosine essentially flat across buckets (0.8563 vs 0.8564)

Two interpretations: (a) variants don't help on macro-policy events because the topic compresses prose toward a common Fed/spread/dot-plot backbone regardless of structural template, OR (b) the n=1 same-variant pair happened to land at the diff-variant mean by coincidence. Cannot distinguish with this sample. Bitcoin's clean −0.05 delta argues for (a) being event-specific rather than the variant approach being fundamentally broken.

## Next steps (drives the verdict)

**Variant approach is alive but underdetermined.** Recommendations in priority order:

1. **Grow the same-variant control set.** Add 2-3 more brokers to fixtures with variant 1, OR rotate through {v1, v2, v3, v1} → {v1, v2, v3, v2} on alternate runs to get same-variant pairs at v2 and v3 baselines too. Goal: get same-variant n ≥ 5 per event so deltas have meaningful confidence intervals.

2. **Investigate fasttrade-pro persona prompt.** The fed-rate fid=0.75 outlier on premium↔fasttrade attributes to forward-guidance framing — a persona-prompt issue, not a variant-prompt issue. Either constrain fasttrade-pro's probability-assertion behavior, or accept that this persona will produce more divergent forward views on macro events. This is independent of structural variants and should be triaged separately before re-running validation.

3. **Re-run on 3-4 events** spanning topic diversity (one rate event, one credit event, one geopolitical event, one crypto event). Two events is the spec floor, not the comfort zone.

4. **Widen Stage 6 to ≥3 identities.** Currently it tests in-house-journalist only — that's one slice of the variant surface. Adding senior-strategist (3 variants, distinct templates) and educator (3 variants, distinct templates) would 3× the test surface for the same persona budget.

5. **Variant 3 specifically looks promising** based on the visual sanity check — it produces the most structurally distinct output (no `#` heading, bold-sentence opener, market-dispatch format). If anything is shippable today, it's that the variant prompts themselves work; the validation surface (1 identity × 4 personas) is too narrow to confirm.

**Do NOT open production-pipeline wiring yet.** Wave 3 verdict is ITERATE, not SHIP — production wiring is gated on a SHIP verdict per the synthetic spec and spec §9 constraint.
